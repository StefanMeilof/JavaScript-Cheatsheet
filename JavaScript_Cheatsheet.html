// 1.1 Variables
Containers for storing data values.
Can contain plain text, numbers, a boolean value or an array
Writen as (var)
// 1.2 Debugging | Alert
On screen message pop-up
Code:
Alert("Text goes here"); =
// 1.2 Debugging | Comments
To test which code does what, you can turn of pieces of your code and see what is controled by those specifie lines of code.
Quick key:
cmd + / (same as html)
// 1.2 Debugging | Console
A built-in tool in Google Chrome to run JavaScript code directly. The Console shows alerts or comments.
Quick key:
Alt + cmd + J, to open console in Google Chrome.
Code:
Use "console.log()" to display JavaScript values in the debugger window.
// 1.3 Data types | String
Javascript uses a string, inside quotiation.
Text between the quotations is used as plain text without any value.
Code:
var Name = "Stefan Meilof";
// 1.3 Data types | Numbers
Javascript recognizes numbers automaticaly, and can use them directly for mathematical purposes.
If you want the numbers to be used as plain text you can put the numbers between quotations
Code:
var Age = 27;

Javascript automatically recognizes numbers and can do math with them.
var x = 4;
var y = 8;
var z = x + y;
// 1.3 Data types | Boolean
Javascript recognizes two possible values: TRUE or FALSE
used for testing arguments.
Example at 1.5 Testing
// 1.3 Data types | Undefined
Javascript doesn't recognize a value
// 1.4 Arrays
Used to store a collection of data in a single variable. It contains a list of elements, these can be with differend datatypes. Can also contain variables.

Array indexes start with 0. So [0] is the first element in the array, [1] the second element, etc.
Code:
var Drink = ["Beer", "Wine", "Cocktails"]
var Holiday = ["Beach", "City", "Snow"]
// 1.5 Testing
See if an argument is True of False using the If and Else statement.
If = outcome true, stuff wil happen
Else if = outcome if-statement not true, test a second argument
Else = (all previous test) outcome not true, other stuff happens

Outcome is always Boolean ("True or False")
Script:
var rand;
rand=5

if (rand > 10) {
alert(rand + "is greater than 10");
}
else if (rand===5) {
alert("Yes, "+rand+" really is equal to 5, so this will show up")
}
else {
alert(rand +" is not greater than 10");
}
// 1.6 Logic
Create a order or sequence for execution of your code. Logical operators are used for boolean logic where a boolean value returns depending on the outcome of an argument. Which can trigger a next argument, statement or (re)action.
// 1.7 Function
A way to encapsulate code for later use. Outcome of arguments can be used as variables inside a function. A value, which can be used later on (or displayed immediately)
